# Baseline styles use layered zero-specificity selectors

sonner-lite ships a minimal visual baseline (white box, border, centered text) in the same stylesheet as the structural CSS, but places those visual defaults in `@layer base` and wraps their selectors in `:where()` so they have zero specificity. It also declares Tailwind's standard layer order (`theme, base, components, utilities`) before the baseline layer, so Tailwind v4 utilities win even if sonner-lite's injected stylesheet is evaluated first. This lets Tailwind v3's unlayered utilities and Tailwind v4's `utilities` layer override the baseline without `!important` or specificity hacks.

Structural rules (positioning, stacking, animations, swipe) deliberately stay unlayered and keep normal specificity so consumer classes cannot accidentally break the mechanics of the toast stack. We chose this split over an `unstyled` prop because "unstyled" would still need structural CSS anyway, and a single layered baseline keeps the public styling surface focused on the documented slots.
